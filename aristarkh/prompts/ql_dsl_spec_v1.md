# QL — Query Language для работы с ИИ (v1)

QL — это специализированный язык описания намерения
для взаимодействия с ИИ и агентами.

Он не предназначен для пользователей “массово”.
Он предназначен для архитекторов и оркестраторов.

---

## Зачем нужен QL

Проблема промптов:
- они неструктурированы,
- плохо воспроизводимы,
- смешивают цель, стиль и детали.

QL вводит:
- явные роли,
- явные ограничения,
- явный формат результата.

---

## Базовая идея

Запрос = структура, а не текст.

QL описывает:
- КТО говорит,
- ЗАЧЕМ,
- В КАКИХ ГРАНИЦАХ,
- В КАКОЙ ФОРМЕ.

## Минимальная структура QL

role: Aristarkh
intent: analysis | design | reflection | teaching
context:

    project: relearning

    user_profile: alexey_profile_v2
    constraints:

    slow

    no_pressure

    no_hype
    output:
    format: markdown | html | code
    size: small | medium | large


---

## Принципы QL

1. Явность важнее гибкости.
2. Ограничения важнее возможностей.
3. Повторяемость важнее “красоты”.
4. Чтение важнее генерации.

---

## Будущее развитие

QL может стать:
- DSL внутри Python,
- конфигурацией для агентов,
- форматом для GUI,
- языком оркестровки ансамблей ИИ.

---

## Важно

QL — это не “язык команд”.
Это язык ответственности.

Если намерение нельзя выразить в QL —
значит оно плохо сформулировано.
